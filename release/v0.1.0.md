# FlowFetch v0.1.0

Initial public release of FlowFetch.

![FlowFetch terminal preview](https://raw.githubusercontent.com/MRT-8/FlowFetch/main/assets/flowfetch-terminal.svg)

## Highlights

- Linux-first CLI downloader for direct file URLs
- Safe archive detection and extraction for `zip`, `tar`, `tar.gz`, `tgz`, `tar.bz2`, `tar.xz`, and `gz`
- Interactive mode for guided use and direct CLI mode for scripts
- Python-first implementation with optional fallback to `curl`, `wget`, `tar`, `unzip`, `gzip`, and `7z`
- Stable exit codes for automation and shell integration

## Included in This Release

- Source workflow with `uv sync` and `uv run flowfetch ...`
- `flowfetch` console entry point from `pyproject.toml`
- Locked dependency set through `uv.lock`
- Linux `x86_64` single-file binary release asset: `flowfetch-linux-x86_64.tar.gz`
- GitHub Actions release pipeline for future tagged releases

## Quick Start With the Linux Binary

```bash
tar -xzf flowfetch-linux-x86_64.tar.gz
cd flowfetch-linux-x86_64
chmod +x flowfetch
./flowfetch --help
./flowfetch https://example.com/demo.zip
```

## Notes

- The release binary is intended for Linux `x86_64`.
- The binary does not require Python or `pip install`.
- Users who prefer the source workflow can continue to use `uv run flowfetch ...` or `python downloader.py ...`.
